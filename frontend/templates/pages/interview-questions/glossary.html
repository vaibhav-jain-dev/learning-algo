Technical Glossary

All technical terms and concepts used across system design topics with detailed explanations and links to where they're used.

Databases & Storage

PostgreSQL

RDBMS

Advanced open-source relational database with support for JSON, full-text search, and geospatial data (PostGIS). ACID compliant with excellent performance for complex queries.

Key Features

  • JSONB: Binary JSON storage with indexing support
  • PostGIS: Geospatial extension for location queries
  • Full-Text Search: Built-in text search capabilities
  • Partitioning: Table partitioning for large datasets

When to Use

  • Complex relationships and joins
  • ACID compliance required
  • Geospatial queries (with PostGIS)
  • Mixed structured and semi-structured data
Used in: URL Shortener Food Ordering Ride Booking Slot Booking

MongoDB

NoSQL

Document-oriented NoSQL database designed for flexibility and horizontal scalability. Stores data in JSON-like BSON documents.

Key Features

  • Flexible Schema: Documents can have different structures
  • Sharding: Built-in horizontal scaling
  • Replication: Automatic failover with replica sets
  • Aggregation Pipeline: Powerful data processing

When to Use

  • Rapidly evolving schemas
  • Document-centric data (logs, content)
  • High write throughput
  • Geographically distributed data
Used in: Notification System

CockroachDB

NewSQL

Distributed SQL database that combines the scalability of NoSQL with ACID guarantees. Designed for global deployments with automatic sharding.

Key Features

  • Geo-Partitioning: Data locality for compliance
  • Serializable Isolation: Strongest consistency level
  • Automatic Sharding: No manual shard management
  • PostgreSQL Compatible: Standard SQL interface

When to Use

  • Global-scale applications
  • Multi-region deployments
  • Strong consistency requirements
  • High availability (99.999%)
Used in: URL Shortener (Global) Food Ordering (Global)

Apache Cassandra

Wide-Column

Highly scalable, distributed wide-column store designed for handling large amounts of data across many commodity servers with no single point of failure.

Key Features

  • Linear Scalability: Performance scales linearly
  • No SPOF: Peer-to-peer architecture
  • Tunable Consistency: Choose your consistency level
  • Time-Series Optimized: Great for time-series data

When to Use

  • Write-heavy workloads
  • Time-series data
  • High availability over consistency
  • Large-scale distributed systems
Used in: Notification (High Traffic)

Caching Technologies

Redis

In-Memory

In-memory data structure store used as database, cache, message broker, and queue. Supports strings, hashes, lists, sets, sorted sets, and more.

Key Features

  • Data Structures: Rich set of data types
  • Pub/Sub: Message broadcasting
  • Lua Scripting: Atomic operations
  • Cluster Mode: Horizontal scaling
  • Persistence: RDB and AOF options

Common Use Cases

  • Session storage
  • Rate limiting
  • Leaderboards
  • Real-time analytics
  • Caching layer
Used in: URL Shortener Rate Limiter Ride Booking

Amazon ElastiCache

AWS

Fully managed in-memory caching service compatible with Redis and Memcached. Provides sub-millisecond latency for caching use cases.

Key Features

  • Managed Service: No server management
  • Auto Scaling: Automatic capacity adjustment
  • Multi-AZ: High availability
  • Global Datastore: Cross-region replication
Used in: URL Shortener (PAN-India)

Message Queues & Streaming

Apache Kafka

Event Streaming

Distributed event streaming platform capable of handling trillions of events per day. Used for building real-time data pipelines and streaming applications.

Key Concepts

  • Topics: Categories for organizing messages
  • Partitions: Parallel processing units
  • Consumer Groups: Scaling consumers
  • Retention: Configurable message retention

When to Use

  • Event sourcing
  • Log aggregation
  • Stream processing
  • Activity tracking
  • Metrics collection
Used in: URL Shortener (Analytics) Notification System Food Ordering

Amazon MSK (Managed Kafka)

AWS

Fully managed Apache Kafka service that makes it easy to build and run applications that use Kafka for streaming data.

Key Features

  • Managed: AWS handles cluster operations
  • Security: IAM, TLS, and encryption
  • Monitoring: CloudWatch integration
  • Serverless: MSK Serverless option available
Used in: Notification (High Traffic)

RabbitMQ

Message Broker

Traditional message broker implementing AMQP protocol. Excellent for complex routing scenarios and when message acknowledgment is critical.

Key Features

  • Exchanges: Flexible message routing
  • Dead Letter Queues: Failed message handling
  • Priority Queues: Message prioritization
  • Acknowledgments: Guaranteed delivery
Used in: Notification MVP

Architecture Patterns

Microservices Architecture

Pattern

Architectural style that structures an application as a collection of loosely coupled, independently deployable services.

Benefits

  • Independent deployment and scaling
  • Technology flexibility per service
  • Fault isolation
  • Team autonomy

Challenges

  • Distributed system complexity
  • Data consistency
  • Service discovery
  • Monitoring and debugging
Used in: Food Ordering Ride Booking

CQRS (Command Query Responsibility Segregation)

Pattern

Pattern that separates read and write operations into different models, allowing optimization for each use case.

Benefits

  • Optimized read and write models
  • Independent scaling
  • Better performance for read-heavy systems
Used in: URL Shortener (High Traffic)

Event Sourcing

Pattern

Pattern where state changes are stored as a sequence of events, enabling complete audit trails and temporal queries.

Benefits

  • Complete audit trail
  • Temporal queries (state at any point)
  • Event replay for debugging
  • Natural fit with CQRS
Used in: Food Ordering

API Gateway

Component

Entry point for all client requests, providing routing, authentication, rate limiting, and request transformation.

Responsibilities

  • Request routing
  • Authentication/Authorization
  • Rate limiting
  • Request/Response transformation
  • Load balancing
Used in: Rate Limiter Food Ordering

Algorithms & Techniques

Token Bucket Algorithm

Rate Limiting

Rate limiting algorithm where tokens are added to a bucket at a fixed rate. Requests consume tokens; if empty, requests are rejected.

How It Works

  • Bucket has maximum capacity
  • Tokens added at constant rate
  • Each request consumes one token
  • Allows burst traffic up to bucket size
Used in: Rate Limiter

Sliding Window Algorithm

Rate Limiting

Rate limiting algorithm that tracks requests in a sliding time window, providing smoother rate limiting than fixed windows.

Variants

  • Sliding Window Log: Store timestamp of each request
  • Sliding Window Counter: Weighted count from adjacent windows
Used in: Rate Limiter

Consistent Hashing

Distribution

Hashing technique that minimizes remapping when nodes are added or removed, essential for distributed caching and sharding.

Benefits

  • Minimal data movement on scaling
  • Even distribution with virtual nodes
  • Fault tolerance
Used in: URL Shortener (Sharding) Rate Limiter (Distributed)

Base62 Encoding

Encoding

Encoding scheme using 62 characters (a-z, A-Z, 0-9) to create URL-safe short codes from numeric IDs.

Why Base62?

  • URL-safe characters only
  • Case-sensitive for more combinations
  • 7 chars = 3.5 trillion unique codes
Used in: URL Shortener

Cloud Services & Infrastructure

CDN (Content Delivery Network)

Infrastructure

Geographically distributed network of servers that cache content closer to users for faster delivery.

Key Benefits

  • Reduced latency
  • DDoS protection
  • Origin offloading
  • Global availability

Popular CDNs

  • CloudFront (AWS)
  • Cloudflare
  • Akamai
  • Fastly
Used in: URL Shortener Food Ordering

Kubernetes (K8s)

Orchestration

Container orchestration platform for automating deployment, scaling, and management of containerized applications.

Key Concepts

  • Pods: Smallest deployable units
  • Services: Stable network endpoints
  • Deployments: Declarative updates
  • HPA: Horizontal Pod Autoscaler
Used in: URL Shortener Food Ordering

PgBouncer

Connection Pool

Lightweight connection pooler for PostgreSQL that reduces database connection overhead and improves performance.

Pooling Modes

  • Session: Connection per client session
  • Transaction: Connection per transaction
  • Statement: Connection per statement
Used in: URL Shortener

Load Balancer

Infrastructure

Distributes incoming network traffic across multiple servers to ensure no single server is overwhelmed.

Algorithms

  • Round Robin
  • Least Connections
  • IP Hash
  • Weighted Round Robin

Types

  • Layer 4 (TCP/UDP)
  • Layer 7 (HTTP/HTTPS)
Used in: URL Shortener Food Ordering
