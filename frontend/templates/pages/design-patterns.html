Design Patterns

Gang of Four patterns with practical implementations

Design Patterns at a Glance
  • Creational: How objects are created (Singleton, Factory, Builder)
  • Structural: How objects relate to each other (Adapter, Decorator, Proxy)
  • Behavioral: How objects communicate (Observer, Strategy, Command)
  • Learn when to use each pattern and see real-world implementations

Creational Patterns

⚫

Singleton

Ensure a class has only one instance with global access point.

Creational
🏭

Factory Method

Define interface for creating objects, letting subclasses decide which class to instantiate.

Creational
🗼

Abstract Factory

Create families of related objects without specifying concrete classes.

Creational
🔧

Builder

Construct complex objects step by step with the same construction process.

Creational
📋

Prototype

Create new objects by copying existing ones (cloning).

Creational

Structural Patterns

🔌

Adapter

Convert interface of a class into another interface clients expect.

Structural
🎨

Decorator

Attach additional responsibilities to objects dynamically.

Structural
🏢

Facade

Provide unified interface to a set of interfaces in a subsystem.

Structural
🛡

Proxy

Provide surrogate or placeholder for another object to control access.

Structural
🌳

Composite

Compose objects into tree structures for part-whole hierarchies.

Structural
🌉

Bridge

Decouple abstraction from implementation so they can vary independently.

Structural
🧳

Flyweight

Use sharing to support large numbers of fine-grained objects efficiently.

Structural

Behavioral Patterns

👁

Observer

Define one-to-many dependency so when one object changes, dependents are notified.

Behavioral
⛰

Strategy

Define family of algorithms, encapsulate each, and make them interchangeable.

Behavioral
📝

Command

Encapsulate requests as objects for parameterization and queuing.

Behavioral
🔄

State

Allow object to alter behavior when internal state changes.

Behavioral
📄

Template Method

Define skeleton of algorithm, deferring some steps to subclasses.

Behavioral
🔁

Iterator

Provide way to access elements sequentially without exposing representation.

Behavioral
👥

Mediator

Define object that encapsulates how objects interact.

Behavioral
🔗

Chain of Responsibility

Pass request along chain of handlers until one handles it.

Behavioral
🚶

Visitor

Define new operation without changing classes of elements operated on.

Behavioral
💾

Memento

Capture and externalize object's internal state for later restoration.

Behavioral
💬

Interpreter

Given a language, define representation and interpreter for sentences.

Behavioral
